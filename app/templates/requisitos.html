Especificação de Requisitos

Versão 1.0

Objetivo do Documento

Este documento descreve os requisitos funcionais e não funcionais do sistema Chatbot PJP, com o objetivo de orientar o desenvolvimento e garantir que o software atenda às necessidades dos usuários e às expectativas dos envolvidos.

Público-alvo

Estudantes do programa Jovem Programador, professores, desenvolvedores envolvidos no projeto integrador e usuários interessados em tirar dúvidas sobre o programa através do chatbot.

Prioridade dos Requisitos

Essencial: O sistema não funciona sem este requisito.

Importante: O sistema funciona, mas de forma limitada sem este requisito.

Desejável: Pode ser implementado em versões futuras, sem prejudicar o funcionamento atual.

Requisitos Funcionais

  • RF01: O sistema deve permitir o envio de perguntas e exibir respostas automáticas via IA.
    Prioridade: Essencial
  • RF02: O sistema deve utilizar raspagem de dados para obter informações do site do Jovem Programador.
    Prioridade: Essencial
  • RF03: O sistema deve exibir uma resposta padrão caso não encontre a informação.
    Prioridade: Importante

Requisitos Não Funcionais

  • RNF01: O tempo médio de resposta do chatbot deve ser inferior a 5 segundos.
    Prioridade: Desejável
  • RNF02: O sistema deve ser acessível por celular e computador.
    Prioridade: Desejável
  • RNF03: O sistema deve estar disponível para consulta 24h por dia, com exceção de manutenções programadas.
    Prioridade: Desejável

Critérios de Aceitação

  • ✅ O chatbot responde perguntas reais com base no conteúdo do site Jovem Programador.
  • ✅ O tempo médio de resposta está abaixo de 5 segundos.
  • ✅ A aplicação funciona corretamente tanto em dispositivos móveis quanto em desktops.
